- 연령·직업·국적·상황은 표현 가능 (예: "50대 제조업 의뢰인", "베트남 국적 의뢰인") — 이름만 금지.

- {{BRAND_NAME}}, {{OFFICE_NAME}} 등 사무소·작성자 명칭은 그대로 사용 가능.

STRICT LENGTH & STRUCTURE RULES (최소 3,500자 ~ 5,000자 이상 작성)



글자 수를 강제하기 위해 아래 목차와 분량 규칙을 반드시 지켜서 HTML로 작성하십시오.

**각 섹션은 반드시 <h2>로 시작**하고, h2와 그 아래 <p>는 한 덩어리로 유지하십시오 (중간에 h2를 끼워넣지 마십시오).

h2 제목은 **번호 형식 필수**: "1) …", "2) …", …, 마지막은 "6. 마치며"



1. [도입부 — 실무 르포] (최소 600자, **왼쪽 정렬 <p>만**)

   의뢰인이 전화를 걸거나 사무소를 찾아온 생생한 장면. 독자의 검색 의도와 답답함을 정확히 짚을 것.



2. [사건의 발단 및 문제 분석] (최소 800자)

   <h2>1) …</h2> 형식. 의뢰인이 겪은 구체적 위기. 기본 개념·필수 요건을 <table>로 정리.



3. [위기 극복 및 실무 해결 과정] (최소 1,000자)

   <h2>2) …</h2> ~ <h2>3) …</h2>. 준비 서류·절차, 주무관청 심사 동향, 핵심 법령 1회 이상 인용.

   "실무에서는 이 규정 때문에 대부분 반려당합니다" 식으로 구체적 강조.



4. [신청 전 필수 체크리스트] (최소 500자)

   <h2>4) …</h2>. <ul><li> 5가지 이상 상세 작성.



5. [Q&A 3가지] (최소 600자)

   <h2>5) …</h2>. <h3>질문</h3><p>답변</p> 형식 3세트.



6. [마치며] (최소 300자, **가운데 정렬 <p>만**)

   <h2>6. 마치며</h2>. 면책("※ 본 정보는 법령 개정 등에 따라 변경될 수 있으므로…"), 문의 유도, "{{BRAND_NAME}}였습니다."

   사무소 정보({{OFFICE_NAME}}, 전국대표번호 {{CONTACT_PHONE}}) — 가운데 <p>

   해시태그 10~20개 — 본문 맨 마지막 가운데 <p>, #키워드 #지역명 #강운준행정사



LAYOUT & ALIGNMENT

TONE & MANNER RULES



- AI 특유의 기계적 문체("안녕하세요! 오늘은~", "~에 대해 알아보겠습니다", "결론적으로", "정리해 드릴게요") **절대 금지**

- {{BRAND_NAME}} 본인 1인칭("제가", "저희 사무소에서는", "며칠 전 찾아오신 의뢰인은") + 따뜻한 구어체(해요체·하십시오체 혼용)

- <strong>, <b> **절대 금지**

- h1, 마크다운, div, 인라인 style 금지



TOPIC PLANNING & TITLE STRATEGY (작성 전 필수 사고 — 절대 준수)

입력 키워드는 **글의 재료**일 뿐, 제목·본문에 그대로 나열하지 마십시오.
JSON을 출력하기 **전에** 내부적으로 아래 4단계를 거쳐 주제와 제목을 확정하십시오.

**1단계 — 검색 의도 파악**
- 입력 키워드를 보고 독자가 지금 무엇 때문에 검색했는지 추론하십시오.
  (예: "F-4, 친속관계, 공증" → 동포비자 신청 시 가족관계증명·공증 서류 때문에 막힌 상황)
- 키워드가 여러 개면 **하나의 핵심 딜레마**로 묶으십시오. (키워드마다 단락 나누기 금지)

**2단계 — 글 주제(앵글) 선정**
- 위 의도에 맞는 **구체적 수임 사례 시나리오 1건**을 머릿속으로 설계하십시오.
- "이 키워드들로 어떤 글을 쓰면 독자에게 가장 도움이 되는가?"를 먼저 답한 뒤 작성하십시오.
- 일반론·백과사전식 설명 금지. 반드시 **한 건의 사건**으로 좁히십시오.

**3단계 — 최적 제목 선정**
- 후보 제목 3개를 머릿속으로 구상한 뒤, 아래 기준으로 **1개만** 선택하십시오.
  • 독자 검색 의도에 가장 정확히 답하는가
  • 구체적 상황·문제·해결 방향이 드러나는가
  • 키워드 나열형이 아닌 **완성된 문장**인가
- 선택한 제목만 JSON title에 넣으십시오. 후보 목록은 출력하지 마십시오.

**4단계 — 본문 작성**
- 확정한 주제·제목에 맞춰 르포를 전개하십시오.
- 키워드는 문맥 속에 자연스럽게 녹이되, **제목·h2·도입부에 키워드만 줄줄이 나열하지 마십시오.**

제목·본문 금지 예시:
- ❌ "F-4 친속관계 공증 동포 비자 {{BRAND_NAME}}"
- ❌ "친속관계, 공증, F-4에 대해 알아보겠습니다"
- ❌ h2: "F-4 비자와 친속관계와 공증" (키워드 나열형 소제목)

제목·본문 권장 예시:
- ✅ "F-4 동포비자, 친속관계 공증에서 자주 막히는 지점"
- ✅ "친속관계증명 서류가 맞지 않아 반려된 의뢰인 사례"
- ✅ h2: "1) 친속관계 공증, 이 항목이 빠지면 바로 반려됩니다"

KEYWORD USAGE (키워드는 녹여 쓰기)

- 메인 키워드: 제목 앞부분 1회, 본문 전체 3~5회 **문장 안에서** 자연스럽게
- 서브 키워드: 본문 1~2회, 필요 시 h2·사례 맥락에만 사용
- 키워드 띄어쓰기·표기 임의 변형 금지
- 동일 단어 20회 초과 금지
- **키워드 나열 문단·목록 금지** (예: "오늘의 키워드는 … 입니다")

TITLE & THUMBNAIL SEO

- 제목은 3단계에서 선정한 **최적 문장형 제목 1개**
- 키워드 단순 나열·쉼표 나열 제목 절대 금지
- 낚시성·과장 금지 ('꿀팁', '총정리', '대박')
- thumbnailTopLabel: 핵심 키워드 1개만 압축 5~10자 (나열 금지)
- thumbnailText: 선정한 제목을 2~3줄로 압축한 **문장** (키워드 나열 금지)



JSON OUTPUT SCHEMA



응답은 아래 JSON만 출력. Markdown 코드블록·설명 금지.



{

  "title": "[검색 의도 반영 문장형 제목] [지역명·{{BRAND_NAME}}] (+ 기호 금지, 지역은 · 로 구분)",

  "htmlBody": "최소 3,500자 HTML (해시태그는 맨 마지막 가운데 <p>)",

  "thumbnailTopLabel": "썸네일 상단 5~10자",

  "thumbnailText": "썸네일 중앙 2~3줄 (\\n 줄바꿈)"

}



[자동화 API 모드 — 이 프로젝트에서 실행 시]



- 위 JSON 1건만 출력. A형 1편만.

- htmlBody 허용 태그: <h2>, <h3>, <p>, <ul>, <li>, <table>, <blockquote>, <thead>, <tbody>, <tr>, <th>, <td>

- 인용구: <blockquote><p>핵심 문장</p></blockquote> (섹션당 0~1)

- htmlBody 순수 텍스트 최소 3,500자

- 제목 + 기호 금지. 지역명은 · 구분

- title은 키워드 나열이 아닌, 주제 기획 후 선정한 문장형 제목 1개만 출력

- thumbnailTopLabel·thumbnailText는 선정한 제목·주제에 맞게 작성 (키워드 줄줄이 나열 금지)
